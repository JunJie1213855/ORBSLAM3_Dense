# ORB_SLAM3_ROS2 — 项目详解

> ROS2 (Foxy) wrapper for ORB_SLAM3 visual-inertial SLAM library.
> 基于 [ros2_orbslam3](https://github.com/curryc/ros2_orbslam3) 修改，增加了 Stereo-Inertial 模式。

---

## 1. 项目概述

本项目将 [ORB_SLAM3](https://github.com/UZ-SLAMLab/ORB_SLAM3)（视觉-惯性 SLAM 算法库）包装为 ROS2 节点，使得 SLAM 系统可以通过 ROS2 的 Topic 机制接收相机图像和 IMU 数据，实时进行定位与建图。

- **包名**: `orbslam3`
- **ROS2 发行版**: Foxy
- **C++ 标准**: C++14
- **编译系统**: colcon + ament_cmake
- **许可证**: MIT
- **测试覆盖**: 无

## 3. 架构设计

### 3.1 总体模式

每种传感器模式都遵循**相同的三层结构**，但各模式之间**没有共享基类**，代码为独立复制：

```
┌─────────────────────────────────────────────────────┐
│                    main() 入口层                      │
│  · 解析 CLI 参数                                      │
│  · 创建 ORB_SLAM3::System（栈分配，指定传感器类型枚举）  │
│  · 创建对应的 ROS2 Node（传入 SLAM 系统裸指针）          │
│  · rclcpp::spin() — 进入 ROS2 事件循环                 │
└────────────────────┬────────────────────────────────┘
                     │ 裸指针 (System 必须比 Node 活得久)
┌────────────────────▼────────────────────────────────┐
│                   ROS2 Node 层                       │
│  · 订阅相机/IMU Topic                                │
│  · 使用 cv_bridge 将 ROS Image → cv::Mat              │
│  · 使用 Utility::StampToSec() 转换时间戳              │
│  · 调用 m_SLAM->Track*() 喂入帧数据                   │
│  · 析构时关闭 SLAM、保存轨迹                           │
└────────────────────┬────────────────────────────────┘
                     │ Track*() 调用
┌────────────────────▼────────────────────────────────┐
│               ORB_SLAM3::System（库内）               │
│  · Tracking 线程：帧间位姿估计                         │
│  · Local Mapping 线程：局部地图优化                     │
│  · Loop Closing 线程：回环检测与全局优化                 │
│  · Pangolin 可视化窗口（若 visualization=true）        │
└─────────────────────────────────────────────────────┘
```

### 3.2 四种模式的差异

| 特性 | Mono (单目) | Stereo (双目) | RGB-D | Stereo-Inertial (双目+IMU) |
|---|---|---|---|---|
| **可执行文件** | `mono` | `stereo` | `rgbd` | `stereo-inertial` |
| **Node 类** | `MonocularSlamNode` | `StereoSlamNode` | `RgbdSlamNode` | `StereoInertialNode` |
| **订阅 Topic** | `camera` | `camera/left` `camera/right` | `camera/rgb` `camera/depth` | `camera/left` `camera/right` `imu` |
| **同步机制** | 无需同步 | `message_filters::ApproximateTime` | `message_filters::ApproximateTime` | **手动同步线程** + `std::mutex` 队列 |
| **IMU** | 无 | 无 | 无 | 有 (200Hz, 累积全部测量值) |
| **图像缓冲** | 不缓冲 | 不缓冲 | 不缓冲 | 仅保留最新 1 帧 |
| **传感器枚举** | `MONOCULAR` | `STEREO` | `RGBD` | `IMU_STEREO` |
| **CLI 参数** | 2 | 3 | 2 | 3 + 1 可选 |

### 3.3 通用工具类: `Utility`

位于 `include/utility.hpp`，只有一个静态方法：

```cpp
static double StampToSec(builtin_interfaces::msg::Time stamp)
{
    double seconds = stamp.sec + (stamp.nanosec * pow(10,-9));
    return seconds;
}
```

将 ROS2 的 `builtin_interfaces::msg::Time`（秒 + 纳秒）转为 ORB_SLAM3 使用的 `double` 秒数。**4 个 Node 都依赖此方法**。

## 4. 各模式详解

### 4.1 Monocular（单目）

**最简单的模式**，约 35 行核心代码。

- 订阅 `camera` topic（QoS=10）
- 每次收到图像，调用 `cv_bridge::toCvCopy()` 转 Mat
- 调用 `m_SLAM->TrackMonocular(image, timestamp)` 进行跟踪

```cpp
// src/monocular/monocular-slam-node.cpp 核心回调
void MonocularSlamNode::GrabImage(const ImageMsg::SharedPtr msg)
{
    m_cvImPtr = cv_bridge::toCvCopy(msg);
    m_SLAM->TrackMonocular(m_cvImPtr->image, Utility::StampToSec(msg->header.stamp));
}
```

### 4.2 Stereo（双目）

- 使用 `message_filters::Subscriber` + `ApproximateTime` 同步策略同步左右图
- 支持**可选的立体校正**：当 `BOOL_RECTIFY=true` 时，从 YAML 读取 `LEFT/RIGHT.K/D/R/P`，用 `cv::initUndistortRectifyMap()` + `cv::remap()` 校正
- 校正与否都调用 `m_SLAM->TrackStereo(imLeft, imRight, timestamp)`

### 4.3 RGB-D

- 使用 `message_filters::Subscriber` + `ApproximateTime` 同步 RGB 和深度图
- 调用 `m_SLAM->TrackRGBD(rgb, depth, timestamp)`

### 4.4 Stereo-Inertial（双目+IMU）— 最复杂节点

这是整个项目**最复杂的部分**，约 220 行实现代码。核心原因：需要手动同步三种异步数据流。

#### 数据流架构

```
ROS2 Topic                        StereoInertialNode
──────────                        ──────────────────
"imu" ──────→ GrabImu()   ──→ imuBuf_ (queue, 累积, mutex保护)
"camera/left" → GrabImageLeft() ──→ imgLeftBuf_  (queue, 仅1帧, mutex保护)
"camera/right"→ GrabImageRight()──→ imgRightBuf_ (queue, 仅1帧, mutex保护)

                    ┌─────────────────────────┐
                    │  SyncWithImu() 线程     │
                    │  (构造函数中启动)        │
                    │                         │
                    │  while(1):              │
                    │    等待三个队列都有数据   │
                    │    对齐左右图像时间戳    │
                    │    (10ms 容差)          │
                    │    收集 ≤ tImLeft 的    │
                    │    所有 IMU 测量值       │
                    │    可选 CLAHE + 校正    │
                    │    TrackStereo()        │
                    └─────────────────────────┘
```

#### `SyncWithImu()` 同步逻辑（关键算法）

```cpp
void StereoInertialNode::SyncWithImu()
{
    const double maxTimeDiff = 0.01;  // 10ms 容差

    while (1)
    {
        // 1. 等待三个队列都有数据
        if (!imgLeftBuf_.empty() && !imgRightBuf_.empty() && !imuBuf_.empty())
        {
            tImLeft  = Utility::StampToSec(imgLeftBuf_.front()->header.stamp);
            tImRight = Utility::StampToSec(imgRightBuf_.front()->header.stamp);

            // 2. 丢弃过旧的右图，直到时间差 ≤ 10ms
            while ((tImLeft - tImRight) > maxTimeDiff && imgRightBuf_.size() > 1)
                imgRightBuf_.pop();  // 丢弃

            // 3. 对称地丢弃过旧的左图
            while ((tImRight - tImLeft) > maxTimeDiff && imgLeftBuf_.size() > 1)
                imgLeftBuf_.pop();

            // 4. 仍无法对齐则跳过此轮
            if (abs(tImLeft - tImRight) > maxTimeDiff) continue;

            // 5. IMU 数据量不够，等待
            if (tImLeft > Utility::StampToSec(imuBuf_.back()->header.stamp))
                continue;

            // 6. 取出左右图像
            imLeft  = GetImage(imgLeftBuf_.front());  imgLeftBuf_.pop();
            imRight = GetImage(imgRightBuf_.front()); imgRightBuf_.pop();

            // 7. 收集时间戳 ≤ tImLeft 的所有 IMU 测量值
            vector<ORB_SLAM3::IMU::Point> vImuMeas;
            while (!imuBuf_.empty() &&
                   Utility::StampToSec(imuBuf_.front()->header.stamp) <= tImLeft)
            {
                vImuMeas.push_back(ORB_SLAM3::IMU::Point(
                    acc, gyr, t));  // 加速度 + 角速度 + 时间戳
                imuBuf_.pop();
            }

            // 8. 可选：CLAHE 直方图均衡 + 立体校正
            if (bClahe_) { clahe_->apply(imLeft, imLeft); ... }
            if (doRectify_) { cv::remap(imLeft, ...); cv::remap(imRight, ...); }

            // 9. 喂入 ORB_SLAM3
            SLAM_->TrackStereo(imLeft, imRight, tImLeft, vImuMeas);
        }
        std::this_thread::sleep_for(1ms);
    }
}
```

#### 设计要点

- **图像缓冲大小为 1**：旧帧被直接丢弃（`pop()`），防止延迟累积
- **IMU 全部保留**：不丢弃任何 IMU 数据，确保惯性约束完整
- **`GrabImageLeft/Right` 在 ROS2 回调线程中执行**，`SyncWithImu` 在独立 `std::thread` 中运行，通过 `std::mutex` 保护共享队列
- **CLAHE 直方图均衡化**：通过 `cv::createCLAHE(3.0, 8×8)` 增强低对比度图像，可选择开启

**编译前必须修改的硬编码路径：**

1. **`CMakeLists.txt:5`** — `PYTHONPATH` 环境变量

   ```cmake
   set(ENV{PYTHONPATH} "/opt/ros/foxy/lib/python3.8/site-packages/")
   ```

2. **`CMakeModules/FindORB_SLAM3.cmake:8`** — ORB_SLAM3 根目录

   ```cmake
   set(ORB_SLAM3_ROOT_DIR "~/Install/ORB_SLAM/ORB_SLAM3")
   ```

### 5.2 CMake 自定义模块: `FindORB_SLAM3.cmake`

从 ORB_SLAM3 构建目录中查找三个库：

| 库 | 查找路径 |
|---|---|
| `ORB_SLAM3` (libORB_SLAM3.so) | `${ORB_SLAM3_ROOT_DIR}/lib` |
| `DBoW2` (词袋库) | `${ORB_SLAM3_ROOT_DIR}/Thirdparty/DBoW2/lib` |
| `g2o` (图优化库) | `${ORB_SLAM3_ROOT_DIR}/Thirdparty/g2o/lib` |

汇总为：
- `ORB_SLAM3_LIBRARIES` = ORB_SLAM3 + DBoW2 + g2o
- `ORB_SLAM3_INCLUDE_DIRS` = ORB_SLAM3/include + ORB_SLAM3/Thirdparty/DBoW2

## 9. 生命周期管理

```
main() 启动
  │
  ├─ ORB_SLAM3::System 构造     ← 栈分配，启动 Tracking/LocalMapping/LoopClosing 线程
  ├─ make_shared<Node>(&System)  ← 创建 ROS2 节点，订阅 Topic
  ├─ rclcpp::spin(node)          ← 阻塞，处理回调
  │     │
  │     └─ 每帧回调 → Track*() → SLAM 线程内部处理
  │
  ├─ rclcpp::shutdown()          ← Ctrl+C 触发
  └─ node 析构 (~Node)
        ├─ syncThread_->join()   ← (仅 stereo-inertial) 等待同步线程结束
        ├─ m_SLAM->Shutdown()    ← 停止所有 SLAM 线程
        └─ m_SLAM->SaveKeyFrameTrajectoryTUM("KeyFrameTrajectory.txt")
                                 ← 保存 TUM 格式轨迹文件
```

## 10. 代码质量观察

| 方面 | 现状 |
|---|---|
| **代码重复** | 四种模式的 Node 高度重复（topic 订阅、cv_bridge 转换、析构清理），但没有共享基类 |
| **硬编码** | PYTHONPATH、ORB_SLAM3_ROOT_DIR、visualization=true、轨迹文件名均为硬编码 |
| **错误处理** | 基本只有 `cerr` + `assert(0)` 或 `RCLCPP_ERROR`，无优雅降级 |
| **ROS2 参数** | 未使用 `declare_parameter` 机制 |
| **测试** | 无任何测试 |
| **线程安全** | stereo-inertial 使用 `std::mutex`，但 `lock()/unlock()` 而非 `lock_guard`，异常不安全 |
| **输出** | 有遗留的调试 `std::cout`（如 "slam changed"、"one frame has been sent"），生产环境应移除或改为 ROS 日志 |

## 11. 扩展方向建议

1. **发布位姿 / 点云到 RViz2**：在 Node 中增加 Publisher，将 `Track*()` 返回的 `Sophus::SE3f` 发布为 `nav_msgs/Odometry`，将 `GetAllMapPoints()` 发布为 `PointCloud2`
2. **提取共享基类**：将通用的 topic 订阅、`cv_bridge` 转换、析构清理逻辑提取到 `SlamNodeBase`
3. **ROS2 参数化**：用 `declare_parameter` 代替硬编码的 `visualization`、轨迹文件名
4. **Lifecycle Node**：迁移到 `rclcpp_lifecycle::LifecycleNode`，支持托管生命周期
5. **Launch 文件**：编写 `launch/*.py` 文件简化启动
6. **TF 广播**：发布 `world → camera` 的坐标变换
